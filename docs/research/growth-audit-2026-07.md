## 1. SEO監査

### 1-1. 内部リンク網【高】

計測結果（src/content/ 全196記事のmarkdown本文を走査）:

- 本文中の内部リンク総数: **227本**（learn/glossary/charts/playbook宛て）
- 本文に内部リンクを1本以上持つ記事: **65/196（33%）**。残り131記事は本文リンクゼロ
  - learn: 21/79、playbook: 10/45、charts: 21/39、glossary: 13/33
- 記事末尾の「同じテーマの記事」（Article.astro）はtopics 13テーマの実データ分類から最大4件を自動表示 → **learn/playbook/chartsは救済されている**が、以下の構造的な穴がある

**構造的な穴（重要）:**

1. **glossaryがリンク網から孤立**。topics.mjs の `topicsOf` は glossary を明示的に除外（`collection === "glossary" ? [] : ...`）。glossary記事は「同じテーマの記事」ブロックが出ず、関連用語（frontmatter `related`）と用語集indexからのみ被リンク。逆方向（learn→glossary）の定義語リンクもほぼない。
   - 例: glossary/drawdown.md 本文は「詳しくは学習記事『ドローダウンと復元の数学』で解説しています」と**テキストで言及するのみでリンクなし**（ライブページでも未リンクを確認）。この形の「言及あり・リンクなし」が用語集に散在。
2. **前後記事ナビなし**。learnは chapter×order で読む順が定義済みなのに、記事ページに「次の記事へ」がない。学習サイトとして読了後の次アクションが「同じテーマ4件」だけ。
3. トップページからの導線は良好（テーマタイル12・学習マップ・レベル別・セクション別4件）。topics 13テーマは記事一覧＋記事末尾リンクの双方向で機能しており、**topicsの設計自体は有効**。

**推奨:**
- (a) ビルド時の用語自動リンク: glossary の term/reading を辞書化し、learn/playbook/charts 本文の初出箇所を `/glossary/◯◯/` へ自動リンク（1記事あたり上限3〜5件・見出し内は除外）。remark/rehypeプラグインで実装可能。逆方向はglossary本文の記事名言及を実リンク化
- (b) learn記事に chapter×order 準拠の前後ナビ（「‹ 前: ／ 次: ›」）を追加
- (c) glossaryにも「この用語が出てくる記事」ブロック（(a)の辞書の逆引きで自動生成）

### 1-2. 表示77からの伸びしろKW【中】

GSCのクエリ内訳は本監査では未取得（要: MasaruのGSC画面確認）。コンテンツ構成からの推定優先KW:

- **「◯◯とは」系（用語集33語）**: 「ドローダウンとは」「だましとは」「アノマリーとは」等は検索意図が明確でAI Overviewsにも拾われやすい。現状のタイトルは用語名のみで「とは」を含まず、この意図に正面から当たっていない（→1-3）
- **チャートパターン名39本**: 「アイランドリバーサル」「セリングクライマックス」等は競合が中規模メディア中心で、図解付き個別ページは相対優位を作りやすい
- **相場格言**: 「頭と尻尾はくれてやれ 意味」等はドメイン強者が少ないロングテール
- 「NISA」「複利」等のビッグワードは当面表示のみでクリック不可の想定。ロングテール（用語・パターン・格言）で初クリックを積む段階

### 1-3. タイトル最適化【高】

- 全ページのtitleサフィックスが「｜投資の学び舎（まなびや）」（**12文字**）。SERPの表示上限（約28〜32文字）に対して重い。41記事はタイトル本体だけで28文字超（最長38文字）であり、サフィックスと合わせるとキーワード後半が確実に切れる
  - 推奨: サフィックスを「｜投資の学び舎」に短縮（（まなびや）はトップ・OGのみ）
- **glossaryのtitleが用語名のみ**（例: 「ドローダウン｜投資の学び舎（まなびや）」）。検索意図「◯◯とは」に不一致
  - 推奨: glossary/[...slug].astro でタイトルをテンプレ生成「`${term}とは？意味と使い方をやさしく解説`」等。frontmatter変更不要でレイアウト1ファイルの修正で完結
- description: 32記事が50文字未満（特にglossary）。SERPスニペットとしては薄く、Googleによる勝手な差し替えが起きやすい

### 1-4. カニバリの有無【高】

タイトル先頭キーワードの重複を機械抽出した結果、**真正のカニバリはほぼなし**。重複14組の大半は「定義（glossary）vs 深掘り（learn/playbook/charts）」「制度解説 vs 手法解説」で検索意図が分離している。ただし以下は要ウォッチ:

| KW | 該当 | 判定 |
|---|---|---|
| ドローダウン／アノマリー／アービトラージ／スキャルピング／だまし／セリングクライマックス | glossary vs learn/playbook/charts | 意図分離可。ただし相互リンク＋glossaryタイトル「とは」化で役割を明示しないとGoogleが同一意図と誤認するリスク【中】 |
| 為替介入 | learn/fx-intervention vs learn/intervention-history | 「仕組み」vs「歴史」で分離。相互リンク推奨 |
| 利益確定 | playbook 3本（格言/ガイド/税金） | kind が異なり分離可 |
| 移動平均線／ローソク足 | learn基礎 vs charts種類解説 | 分離可・相互リンク推奨 |

対処は「統合」ではなく「相互リンク＋タイトルでの意図明示」で足りる。

## 2. AIO監査（AI検索最適化）

### 2-1. 構造化データ【高】

現状は Article.astro の1種のみ:

```json
{ "@type": "Article", headline, description, dateModified, inLanguage, publisher }
```

不足（すべてビルド時に既存frontmatterから機械生成可能）:

1. **glossary: DefinedTerm が未実装**。`term`・`reading`・`description`・`related` が構造化済みでスキーマ化コストが最小なのに、AI・検索エンジンには通常記事に見えている。`DefinedTerm`（+ index に `DefinedTermSet`）を追加すべき。33語→用語集は今後の量産対象でもあり先に配管を敷く価値が高い
2. **BreadcrumbList**: 全ページ未実装。ホーム›セクション›記事 の3階層をレイアウトで機械生成可能
3. **Article拡張**: `datePublished`（現状dateModifiedのみ）・`image`（OGP画像パスが既にある）・`mainEntityOfPage`・`author`（Organization=編集部で匿名運用と両立）
4. **FAQPage**: 現状FAQ形式の本文がないため対象なし。将来「よくある質問」セクションを記事に足す場合のみ検討（コンテンツ変更を伴うためCEO判断）

### 2-2. AIに引用されやすい定義文・要点【高】

- glossary冒頭は「資産残高の**ピーク（最高値）からの下落率**です。」のように**主語を省いた定義文**。AI抽出・強調スニペットには「**ドローダウンとは、**資産残高のピーク（最高値）からの下落率です。」と主語込み1文完結が有利。テンプレ規約として今後の記事に適用し、既存33語は順次改修（コンテンツ変更のためCEO承認）
- learn記事は冒頭「## 結論」セクションが要点ボックスの役割を既に果たしている【高・良い点】。この構造はAI引用と相性が良く維持すべき。視覚的な要点ボックス化（枠付きCallout）はデザイン変更として任意

### 2-3. llms.txt【高】

- `https://toushi-manabiya.jp/llms.txt` は**404**（ライブ確認済み）
- 推奨: サイト概要・編集方針（出典A〜E格付け）・主要セクションURL・topics 13テーマ一覧を記載した llms.txt を public/ に設置。196記事のインデックス（llms-full.txt またはセクション別リンク）も自動生成可能。RSS・sitemapがある現状でも、方針と構造を1枚で示す効果は大きい【中】

## 3. UX監査

### 3-1. 学習動線【高】

- 初心者の入口は「学習マップ（/learn/）」＋トップの学習マップ・ミニチュア。7章×4レベルの体系は明快
- **欠落: 「はじめての方へ」ページがない**。「知識ゼロの初回訪問者が最初の1記事を選ぶ」判断をヒーロー検索カード（＝キーワードを知っている前提のUI）に委ねている。求人ポータル型検索カードは再訪者・目的検索には強いが、初訪の無目的層には「何を検索すればいいか分からない」問題がある
  - 推奨: /start/（はじめての方へ）を新設し、「最初の10記事」を読む順に提示（Lv.1入門から章順）。トップのヒーローに小さく「はじめての方はこちら」導線を追加。→ デザイン・構成変更
- ヒーロー統計「60本+ 執筆予定」が公開196本と矛盾（公開数が予定数の3倍超）。信頼の単位としての「正直な数字」が逆に古さの印象を与える。数値の更新または文言撤去を推奨（軽微・即実装可）

### 3-2. 記事内回遊【高】

- 「同じテーマの記事」4件＋テーマ一覧リンク: 機能している
- 前後記事ナビなし（→1-1b）。学習サイトの中核回遊装置として最優先
- 記事ページ上部にパンくず・カテゴリ/レベルバッジがなく、検索流入者が「どの章・どのレベルの記事か」を把握できない。topicsページには実装済みのメタバッジ群を記事ヘッダにも出すと文脈が伝わる

### 3-3. 検索カードの機能性【高】

- キーワード入力→Pagefind検索、章/レベル選択→一覧遷移のハイブリッドは実装として妥当。Pagefind UIの日本語化も丁寧
- 「人気の検索」5語がハードコード。GSC/GA4の実クエリが貯まったら実データ由来に差し替えるべき（tone.mdの「根拠なき人気表示をしない」原則との整合も取れる）【中】
- 用語集indexは五十音ソートのみでジャンプナビ（あ・か・さ…）なし。33語では許容だが、用語量産前にア行ジャンプを入れておくと拡張に耐える
